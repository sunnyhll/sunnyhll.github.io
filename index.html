Authentication服务鉴权机制

用户在访问系统或者服务时，服务器端需要验证用户是否拥有访问的权力，这个过程称为鉴权。在服务器-客户端架构的软件系统中，当一个没有经过鉴权的用户登录时，服务器可能会返回鉴权请求。鉴权是一种客户端和服务器协同认证的方式，有多种方式可以实现：

  • HTTP Basic Authentication
  • session-cookie
  • Token 验证(JWT)
  • OAuth(开放授权)

HTTP Basic Authentication

HTTP认证时一种无状态的认证模式，因此用户在提供相关凭据的请求中能够得到认证用户的访问，而服务器本身在接下来的请求中并不能持续保持登录状态。

HTTP基本认证的具体流程如下:

在你访问一个需要HTTP Basic Authentication的URL的时候，如果你没有提供用户名和密码，服务器就会返回401，返回Header中会包含类似”WWW-Authenticate: Basic realm=”test””信息。如果你直接在浏览器中打开，浏览器会弹出对话框提示你输入用户名和密码。

要在发送请求的时候添加HTTP Basic Authentication认证信息到请求中，有两种方法:

  1. 在请求头中添加Authorization信息：Authorization: “Basic {用户名:密码}的base64加密字符串”。
  2. 在url中添加用户名和密码。

其中，鉴权机制中的身份验证并非一定要依赖Basic方式的用户名密码作为凭据，可以通过如下方式：

  1. “WWW-Authenticate: Negotiate” SPNEGO协议,支持Kerberos, NTLM点对点认证方式完成。在协商过程中：

    • 可以请求 Authorization: Negotiate {kerberos票据}进行Kerberos验证
    • 或者也能读取返回头中的 Authorization: Negotiate NTLMSSP{八字节质询码} ，并在请求头部中加入 Authorization: Negotiate NTLM{加密的质询码和明码用户名}

  2. “WWW-Authenticate: Digest realm=”test”,qop=”auth”,nonce=”{md5加密时间},opaque=”{不透明字符串}”摘要认证协议，能避免明文传输数据。

    • 在请求中需要加入 Authoriztion: Digest username=”guest”,realm=”test”,nonce=”{同上},qop=”auth”,nc=”00000001”,response=”{通过md5加密的user paswd httpmethod uri等信息}”,cnonce=”{客户端提供的非明文字符串}”,uri=”{uri信息}”
    • 服务器需要检查时间在允许范围内，而且response匹配本地生成值。使用MD5算法的优势在于可以很快正向哈希，而无法短时间内逆向哈希得出用户密码等信息。

session-cookie鉴权

Session是HTTP协议中为了支持有状态的通信而发明的会话机制，本质上是通过服务器为用户建立sessionid从而保证用户的状态信息能够在服务器端保存。用户不需要反复进行登录认证就能保持会话。

而Cookie则是一种特殊的HTTP头部，能够在HTTP通信中保存一定的用户信息，如sessionid从而达到认证用户的目的。由于Cookie本身是针对某一域名而产生的，所以在发送Cookie过程中必须提供正确域名的sessionid才行。

Token 验证(JWT)

Token验证方式和Seesion验证方式很类似，不同的是Token本身包含一些有意义的信息：用户名、密码、过期时间等。Token本身由服务器签发，客户端请求的发送中需要包含 Authorization : JWT “{jwt token}”，服务器提取token信息通过相同的算法验证即可。相较于Session验证方式节约了分布式系统中服务器存储sessionid和用户信息的开销，只需要服务器拥有相同的密钥即可。

JWT(json-web-token)算法细节：

JWT由三部分”{header}.{payload}.{signature}’,两种算法生成，公式如下：
signature = sha256(base64(header)+’.’+base64(payload),{服务器密钥})

  1. header包含算法和类别信息,
  2. payload为加密部分，包含公有声明和私有声明，公有声明为约定的key，私有为公司定制key，
  3. signature，算法签名。
  4. sha256为header中写的加密算法，基于服务器密钥生成不同的加密签名,具有不可逆性
  5. base64为编码算法，可逆运算

OAuth2/OIDC认证

OIDC 即Open ID Connect, 是一种基于OAuth2授权流程，并且扩展了身份认证层的一种新的认证机制。

OIDC认证模型主要包含如下四个角色和一个令牌（完整术语参见http://openid.net/specs/openid-connect-core-1_0.html#Terminology）：

  • EU用户：End User：一个人类用户。
  • RP客户端：Relying Party ,用来代指OAuth2中的受信任的客户端，身份认证和授权信息的消费方；
  • OP认证服务器：OpenID Provider，有能力提供EU认证的服务（比如OAuth2中的授权服务），用来为RP提供EU的ID Token身份认证信息和Access Token访问令牌；
  • UE用户资源服务器：UserInfo Endpoint用户信息接口（受OAuth2保护），当RP使用Access Token访问时，返回授权用户的信息，此接口必须使用HTTPS。
  • ID Token认证令牌：JWT格式的数据，包含EU身份认证的信息。通过OP提供。

OIDC的支持的授权流程如下：

  1. Authorization Code(授权码模式)：使用OAuth2的授权码来换取Id Token和Access Token。
  2. Implicit (简化模式)：使用OAuth2的Implicit流程获取Id Token和Access Token。
  3. Hybrid(混合模式)：混合Authorization Code +Implicit。

OAuth2授权模型

OAuth2的授权模型时为了已登录用户通过第三方应用访问资源服务器进行授权的流程，授权模型和OIDC相似，包含如下四个角色:

  • 资源拥有者（User） - 指应用的用户，比如github的一个账户拥有者
  • 认证服务器 （Authorization Server） - 提供登录认证接口的服务器，比如：github等
  • 资源服务器 （Resources Server） - 提供资源接口及服务的服务器，通常和认证服务器是同 一个应用。
  • 第三方客户端（Client） - 第三方应用，希望使用资源服务器提供的资源，比如你的一个支持通过github账户登录的应用
  • 服务提供商(Provider): 认证服务和资源服务归属于一个机构，该机构就是服务提供商，比如github公司

OAuth2具有四种授权模式，下文将分述这四种模式具体流程：

  • 授权码模式（authorization code）
  • 简化模式（implicit）
  • 密码模式（resource owner password credentials）
  • 客户端模式（client credentials）

授权码模式(最为常见)

  1. 用户访问客户端应用
  2. 引导用户到认证服务器进行登录（此步骤需要携带客户端应用的clientId，可以是html直接转发认证服务器），用户输入用户名、密码
  3. 认证成功后，认证服务器向客户端应用发一个授权码code
  4. 客户端应用拿着授权码code，和clientId，clientSecret，去换取access_token
  5. 返回access_token给客户端应用

这种场景下，用户名、密码、客户端应用信息，都没有直接暴露在浏览器，是web下是最安全的。

简化模式

授权码模式的简化，用户认证成功后，直接将token返回给浏览器。因为某些应用没有前端服务器，只有一堆静态的html（很少见），这种模式，一般不用。

密码模式

适用场景：手机app ，这个客户端应用是你完全可以信任的，你的app就是自己公司开发的。但是这个模式并不适合在web场景下用，在web下，用户名密码并不是直接填给自己写的应用的，而是填在浏览器呈现的一个页面上的，这个浏览器是客户端应用的一个代理，浏览器是没法保证安全性的。

客户端证书模式

客户端应用直接发 clientId、clientSecret给认证服务器，发的令牌是针对客户端应用的，不是针对用户的。跟没授权一样，令牌不能识别用户身份。

Authentication实战

本章将着重描述如何在java springboot应用中实现相应的认证流程。springboot提供了一站式应用的开发模式，但是认证流程是需要spring security，同时具体的认证核心模块需要spring securty keberos或者spring security oauth组件支持。下文将主要介绍如何利用这两个模块实现具体的基于siteminder/spnego/oauth协议的认证流程。

siteminder sso + preauth

Siteminder是企业级认证产品，它提供了一站式认证中间件，从应用开发者的角度来看，就是采用了外部认证系统，应用不需要重新进行认证而是可以直接从siteminder处理过的http request header中提取SM_USER中拿到userprincipal()。因此从spring securty框架的角度之需要直接读取认证后的信息，而不需要再对request进行认证验证。这通常是一种企业内网用户认证采用的sso机制，因为用户之需要进行简单认证后就能得到对多种内部webapp的访问toekn，而且不需要进行细粒度的鉴权的场景适合大部分内部应用，但是，因为外部网站可容易会被假的header所欺骗，安全性较差而不会采用这种方式进行验证。

spnego auth

SPNEGO是微软设计的一种企业级认证协议，底层支持多种token协议，因此是应用proid常用的一种方式，因为应用的id会常常跑在不同的window/linux环境，而spnego能够支持多种密钥认证从而对跨系统认证能有很好的支持，这种认证方式需要用户自己执行认证检查，所以需要spring-security-keberos模块的相关auth provider进行验证。

oidc oauth2 + ping federate

oauth标准的实现一般是用oidc协议，spring security oauth2拥有对oauth2标准的鉴权模型的实现，并且可以通过适当的配置完成oidc用户验证。oauth单独并不能完成验证鉴权功能，需要部署一个oauth provider例如云服务商azure AD等，企业级内部可以自己部署ping federate服务器完成auth信息提供功能，并且向下兼容sso（即open Id）功能。

窗口消息

GUI应用程序必须响应来自用户和操作系统的事件。

来自用户的事件包括用户与程序交互的所有方式：鼠标点击、按键、触摸屏手势等等。
来自操作系统的事件包括程序之外的任何可能影响程序行为的东西。例如，用户可能插入一个新的硬件设备，或者Windows可能进入低功耗状态(睡眠或休眠)。
这些事件可以在程序运行时的任何时间发生，几乎可以按任何顺序发生。如何构造一个不能预先预测执行流（flow）的程序?

为了解决这个问题，Windows使用了消息传递模型。操作系统通过传递消息与应用程序窗口进行通信。消息只是指定特定事件的数字代码。例如，如果用户按下鼠标左键，窗口将接收到具有以下消息代码的消息。

C++

#define WM_LBUTTONDOWN 0x0201

一些消息具有与它们相关联的数据。例如，WM_LBUTTONDOWN消息包含鼠标光标的x坐标和y坐标。

要将消息传递给窗口，操作系统将调用为该窗口注册的窗口过程。(现在你知道窗口程序的作用了。)

https://learn.microsoft.com/zh-cn/windows/win32/learnwin32/your-first-windows-program

窗口句柄

窗口句柄是消息传递的源和目的，在每一个进程空间窗口句柄都是局部的，需要通过窗口名字去发现，对于某个窗口消息队列的监测，可以通过visual studio自带的spy++去定位监听并且调试。

非暴力沟通四要素

  • 首先，留意发生的事情，清楚地表达观察结果，而不判断或评估。
  • 接着，表达感受，例如受伤、害怕、喜悦、开心、气愤等等。
  • 然后，说出哪些需要导致那样的感受。
  • 最后，提出非暴力沟通的第四个要素——具体的请求。这一要素明确告知他人，我们期待他采取何种行动，来满足我们。

观察误区

  • 道德评判：对他人的评价实际上反映了我们的需要和价值观。应该直接说出我们的价值观，而不会指责他人。
  • 比较：比较也是一种评判。
  • 回避责任： 我们可以用负责任的语言代替回避责任的语言。补充我为什么要这么做，以及自己的感受。
  • 强人所难：我们无法强迫他人按我们的期待生活。认为“某人应当受到惩罚”使我们难以体会到心中的爱。

    7. 相互依存（Interdependence）

请求帮助

请求别人帮助时候，需要更具具体的目的而做出不同的请求：

  • 提出具体而且清晰的请求
  • 明确谈话的目的
  • 请求反馈能确保对方准确把握我们的意思
  • 参加集体讨论时，说清楚我们希望得到怎样的反馈
  • 请求没有得到满足时，提出请求的人如果批评和指责，那就是命令；如果想利用对方的内疚来达到目的，也是命令。

用全身心倾听

在倾听他人的观察、感受、需要和请求之后，我们可以主动表达我们的理解。如果我们已经准确领会了他们的意思，我们的反馈将帮助他们意识到这一点。反之，如果我们的理解还不到位，他们也就有机会来纠正我们。此外，这样做还有助于人们体会自己的状况，从而深入了解自己。

当对方得到充分的倾听，（a）我们将体会到气氛变得轻松；（b）他会停止谈话。

当我们痛苦得无法倾听他人时，我们需要（1）体会自己的感受和需要（2）大声地提出请求（3）换一个环境。

倾听的力量

倾听可以使我们了解他人语言背后的感受和需要，能够更好的与他们坦诚沟通。

  • 倾听可以辅助提醒对方了解自己的内心感受。
  • 倾听是建立信任感的有效手段。

爱自己

非暴力沟通最重要的应用也许是培养对自己的爱。

  • 非暴力沟通的忧伤：当我们的行为无法满足自身的需要，我们体会着人生的悲哀和内心的渴望。

  • 非暴力沟通自我宽恕：感到遗憾时，我们试图了解过去的行为所要满足的需要。

  • 让我们的行动基于对生命的爱，而非恐惧、内疚、羞愧或义务。

  • 不论你选择做什么，了解自己为什么要那样做。”我选择做______是因为我想______”

充分表达愤怒

  • 生气的原因在于我们的想法——对他人的评判和指责。希望他人因为内疚发生改变，就是将刺激和原因混为一谈。我们无须压抑愤怒，只要我们专注于他人的感受和需要，愤怒也就不再存在。

  • 愤怒驱使我们去惩罚他人。留意头脑中出现的暴力想法，而不评判它们。

  • 当我们意识到自己的需要，愤怒就转变为服务需要的情感。职责别人没做的事情，往往是我们想要对方做到的事情，而这件事情会满足我们的某些需求。

表达愤怒的步骤：

  1. 停下来。呼吸。
  2. 留意我们的指责。
  3. 体会我们的需要。
  4. 表达感受和尚未满足的需要。

运用强制力避免伤害

使用防卫性的强制力，是为了保护自己或他人，而不是为了惩罚、羞辱或谴责他人。

此时，我们希望痛苦能让他们：

  1. 意识到自己的过错；
  2. 感到懊悔；
  3. 改变行为。

然而，在实际生活中，惩罚往往加强了对方的敌意和抵触心理，使双方的关系更加疏远。

有两个问题可以帮助我们认识惩罚的局限性。

  • 第一个问题是，如果我不喜欢他现在的行为，那我希望他怎么做？如果我们只问这个问题，惩罚可能是有效果的，因为威胁或惩罚可以对他人的行为产生影响。
  • 然而，通过第二个问题，我们就可以清楚地看到惩罚的局限性：我希望他基于怎样的原因去做我想要他做的事情？

重获生活的热情

  • 认识情感的根源在于个人的需要和想法，并以建设性的语言提出明确的请求
  • 专注于我们想要做的，而不是追究错在哪里。
  • 体会自己的感受和需要可以释放压力。
  • 体会他人的感受和需要也可以释放压力。

表达感激

用非暴力沟通的方式表达感激时，我们只是为了庆祝他人的行为提升了我们的生活品质，而不是想得到任何回报。

  1. 对方做了什么事情使我们的生活得到了改善；
  2. 我们有哪些需要得到了满足；
  3. 我们的心情怎么样？

在别人表达感激时，人们通常有两种截然不同的反应。一种是自我膨胀，相信我们比别人优越；另一种是假谦虚，否定别人的欣赏，耸耸肩说：“哦，这没什么。”

《领导力》读后感

成为领导者的基本能力

领导者是需要一种成就他人，成就团队的能力，相较于个人自身能力，反而不是最重要的。本书的作者是一名职业足球教练，从职业足球选手退役后，变成了一名带队教练，积累了多年的经验并且成功成就了一支超强的足球队伍，总结他的教练生涯上成功的关键因素。

  • 聆听

    • 聆听是最直接的了解对方和自己的方式，成功者的经验和话语或多或少反应了他的价值观，也对自己会有一定的启发，没有利益冲突人的忠告往往是真诚和有用的，要自己思考和体味其中价值并未自己带来改变。

  • 观察

    • 细节观察：对于有重要影响力的细节，需要着重观察，但是不能深陷细节的泥淖，丢失全局的目标。

    • 整体观察：局势，核心要素，评估对手，是整体观察的重要因素，5W准则，回答Why，What，Where，Who，When问题。

  • 阅读

    • 对于你关注的领域，永远需要孜孜不倦的热忱去提升你对领域的认识，外界环境可能会对你的汇报不是同侪对等的，但正确的努力方向，一定会带给你想要的汇报。万物的哲学规律是相同的，不要局限于同意领域的经验，有时候新的领域的成功经验也会带给你新的思路。

领导者青睐的对象

领导者和成功的团队之间是有相互吸引的作用力的，即使会有小的嫌隙，但总的方向上，一定是默契的。

  • 纪律

    • 纪律性是一支队伍需要有有种一致的信念，对于不遵守纪律的队友，在权力范围内，可以更换掉，当然在惩罚力度上需要合适，避免过于严苛打消队伍的积极性。足球俱乐部的惩罚力度分为，罚工资，坐冷板凳，停赛和转会，职场上也是类似，言语警告、罚奖金、减少项目机会，没有新项目和转组换公司。惩罚的后果除了提振队伍纪律，也会让队伍丧失一些机会，也是团队领导者需要承受的损失。

  • 工作效率

    • 不想当将军的士兵不是好士兵，求胜欲和上进心是一个队员成功的必须条件，努力努力再努力是内心的渴望的一种体现，无论是童年的苦难还是内心的渴求驱使，一个人永不言弃的追求是非常宝贵的。这种渴求会让队员拥有超乎寻常的刻苦，成就超人的效率，不是因为拥有单位时间更加高的天赋，而是更多的时间精力投入收获了更多的效果。

  • 能动性

    • 能动性是一个人意志力的表现。永不服输，渴望成功的心情是每个人都有的，但是意志力随着环境、时间推移，不变的人是很少的，超凡的意志力能够让一个人持续进取，持续输出，构建并实现出宏伟的蓝图。

  • 信念

    • 足球是团体竞技体育的典型代表，个人力量是无法支配比赛的，因此球员的信心在漫长的90分钟比赛期间的起伏也会影响比赛的结局，哀兵必胜，但是萎缩的信心是永远无法带来胜利的。

领导者对团队的建设

足球比赛就像一场战役，队员用血肉之躯去竞技，当中会有心情浮动，体力消耗，环境因素，早就一支长盛不衰的队伍是需要有意图的训练，并且临场灵活的调度。

  • 组织：团队的组织架构和信仰是持续稳定的，相对来来走走、个性不同的球员，团队的组织架构能够带来良好的运营与稳定性。

  • 准备：团队赛前准备充分，就能灵活应对赛场上的灵活多变，战术上的充分准备能够让队员更加理性的看待战况，打出自己的水平。

  • 管道：球队的运营和公司很像，在流动的队伍中创建持续的质量输出，培养球员，高薪聘请，运动员的运动生涯周期，都会影响球队的成绩。类似公司的职级架构，球队的组织层次分为训练营、B队、A队、预备队和首发阵容。培养的重心也会从初级的技术体能转而向情感和心理需求管理。球员之间的情谊也是很重要，这对团队建设也是非常重要，有时候能够起到一加一大于二的效果。

构建吸引人的团队

  • 团队合作：团队平均年龄是管理者考量的一个因素，平衡的结构有助于队伍保持高速发展，不是说年纪大了就没有用，但是对于运动员而言，年纪大了不够核心被迫选择离开也是两难的境地。领导者不要因个人喜好去选择人才，而是需要根据团队需求去吸收希望有的人才，构建核心团队。

  • 队长：需要具备如下四个特质：

    • 有领导欲望：
    • 能够传达教练意愿：
    • 队伍中具有威信
    • 能够随机应变：

领导者如何评价团队队员的水平

  • 优秀的榜样：任何领域都会有榜样，世界级的榜样屈指可数。世界会有完备的评分体系，去评估每一位队员的个人能力，不难发现优秀人才。给团队指定合理的目标能最大化进展，太高或者太低目标都不能达到最大化效果。

  • 鼓舞：队员潜力开发需要鼓舞，不需要和队员打成一片，适当的敬畏是需要的，这是指挥者必须要遵守的原则。对于每个队员的激励因素是不一样的，而且不要随便变化！教练的动力就是获得冠军，队员都需要为这个目标去努力。对于有效的鼓励和监督场景，领导者不要缺席。

  • 自满：自满情绪会造成失败，无论对领导者还是队员，哀兵必胜。即使现在已经有成就，吸引人的永远是下一个冠军就。

领导对人才的态度

  • 求职面试：领导面试求职者时，希望发现发现面试者有何雄心、是否只是把工作当作跳板。除了素质和资历，充实程度也很重要。领导需要寻找的时热情、积极态度、眼神的交流和个人的勇气。

  • 人际关系网：在职场中，人际关系网非常重要，值得花时间去培养，别人的建议往往能够拨开迷雾。当然人机关系网需要有方向的建立，足球队教练的人际关系网，除了同侪，现役球员，还有前任球员，球队支持者。对于普通职场的人，除了领导，下属，团队之外，没有绝对利益相关的同事也可以成为人际关系网的一部分，甚至在公司外部，也需要关系网，人不是孤独的一座小岛，人生漫漫长路一定需要互相扶持帮助。

  • 炒鱿鱼：职业运动生涯的淘汰率时很高的，避免不了严格的淘汰机制，解雇的场面一定时尴尬和令人难受的。尽可能的帮助别人找到合适的领域，也是一份绵力。

领导的自我诉求

  • 时间：每个人每天都只有24小时，能人是如何具有超乎常人的产出？一定是超出常人的付出，更多的时间花费在自己想要深耕的领域，长久下来就会有变化。

  • 分心：专注在一个领域对于喜欢这个领域的人而言，不是大问题，把精力集中在重要的事情上，建立良好的个人特质，有利于长期的发展。近现代的生活充斥太多令人分心的东西，让人眼花缭乱，这些事情会让时间耗费在不留痕迹的投入中，并且得不到任何的收益。对于足球运动员而言，读书也是一种分心，当然对普罗大众而言，读书的必须的。

  • 失败：积极态度面对失败，每个人都会有失败不要害怕。人对于生存下去的渴望是有本能的，避免失败，就能成功。对于顶级足球教练而言，球队在赛季的胜率也就不过72%，也就说再优秀的人也面临过失败。在失败中从自身找问题，能够更好的利用失败成长。同时，失败也让我们知道了自己的弱点，和优势，并且同时知道如何调整自己的方向和展示角度。

  • 批评：批评对于批评者而言，是容易的，因为他不关心执行过程中的细节和微妙心态，只需要专注于找到不足之处。而对于执行者而言，面对批评的压力，正确的看待是有益处的。即使最坏的事情发生了，也不会是世界末日。当然再强硬的人也会有软弱的时候，这段时期，需要重新建立信心，和主动和自己和解。

领导力传递

  • 说：跟各种人打交道，不同的人需要采取不同的方式。养成换位思考的习惯，把自己想象成聆听者，关注技艺精湛的公共演讲者。

  • 写作：作者出书，我第一本书《北部之光》（A Light in the North）,第二本书是《管理我的生活》（Managing My Life），忙里偷闲时胡乱写下自己的想法和记忆的好习惯能够有助于写作。

  • 回答：与媒体打交道可以呈现出不同的形式，但是其中的关键就在于控制。情感和肢体语言不要受到环境的影响。

领导与管理的不同

我的工作是制定非常高的目标。我的工作是让人们相信自己能够做到之前认为自己做不到的事情。我的工作是画出一条之前从未走过的道路。我的工作是让每个人都相信世界上没有不可能。这就是领导和管理之间的区别。

  • 对老板汇报：对于那些你需要进行汇报的负责人，了解其性格是何等重要。

  • 控制力：无法控制航线、调整航速的船长是不能够安全抵达港口的。完全控制是有效领导的必要条件，但是细节和micro管理是两回事。

  • 授权：随着事业的发展，其团队成员也越来越多。这就要求对其周围人员进行授权。如今所有的大型俱乐部都有首席执行官，由其负责全部的商业活动并确保账目平衡，这对于很多俱乐部来说，不至于太过失控。因此我让大卫·吉尔负责电视合同、拉拢赞助商、敲定球员合同的细节、管理财政和市场营销部、应付稽查员和律师、确保俱乐部遵守健康与安全条例以及适用于所有集团的法律法规。领导者在乎细节，而具体工作需要他们来做。

  • 决策：有效的授权依靠的是其他人做出决定的能力。做出决定所依据的是你目前可以使用的信息，并且在正确的时机。

销售思维

领导者需要擅长销售自己的想法，说服下属或者对手方去做他们不想做的事情。作者作为足球教练，主要在如下几个专项上施展自己的领导力。

  • 买球员: 教练为了组建一支超强队伍，需要不断去搜寻和购买具有潜力的年轻队员，就像职场中，也需要可以培养良好的梯度，为了队伍的正常新老更新做准备。

  • 节约：总教练节约，可以为队伍日常开支进行一定的节约。

  • 报酬：球员的报酬，除了考虑个人能力，也要考虑同侪效应，整体蛋糕切分尽量公平为了整体稳定性。

  • 谈判：和下属谈判，是领导不可避免的问题，确立好目标和底线，在平稳情绪过程中进行，保证谈判结果的合理性。

  • 经纪人：经纪人作为领导者和潜力下属的中间人，也是不可避免的角色，了解经济活动作为中间人的特性，能够良好的避免其带来不良的影响。

技术与信息

技术在新时代为球队，球员的培养提供了重要的指标和导向。现代化的队伍建设量化了每个球员的特性，科学化培养成了标准流程，让领导对于每一个下属的培养有了更加标准化流程，不再需要发挥个人特性，熟悉流程可以合适培养大多数的人才。

  • 创新：技术革新和科技理论的更新创造了新的培养队伍方式，领导者需要紧跟时代进行灵活升级。

  • 数据过载：现在的传感器技术已经能够对每一个小事进行细节捕捉，然而这样也造成了数据太多的问题，不要完全淹没在数据海洋里，有时候人的经验和直觉往往不需要海量数据的挖掘，也会更高效。

  • 机密：队伍成功是必须要有保密守则，对于领导注重的信息，不能擅自泄露，对于违反规定的人，必需找机会以提醒甚至惩戒。

竞争与环境

  • 对手：对手的不理智行为，往往会影响决策者的正确性，同时也容易陷入博弈论的恶性循环中，需要理智并且妥善化解。

  • 全球市场：全球化的人才市场为雇佣关系带来了更加充分的市场竞争，有助于企业和队伍融合成长。

全新的挑战

挑战是分层次的，当你准备往前进一步新的挑战时，需要妥善处理自己的现有权力，将其分担给合适的人选。

  • 新官上任：新官上任三把火，可以建立威信，急于展示自己的领导者可能会急于求成做出过激的表现。

  • 离开：职位的交接，每一个职位都有它固有水平的池子，池子越小不代表更加不好，而是横向流动的机会一旦产生需要尽快把握住，不然也需要很长的等待期，如果被降级也不一定有更好的机会流回最佳的位置。

  • 新的节奏：到了新的生活阶段，即使一个成功的领导者也需要适应新的节奏，培养新的兴趣爱好来填满自己的生活。

《走对小学的每一步》陪伴，为孩子打下扎实基础 读后感

一年级，培养控制力

习惯力：教育就是养成良好的习惯

  • 时间习惯：把时间“主权”还给孩子
  • 学习习惯：写作业不再“鸡飞狗跳”
  • 阅读习惯：让孩子任性去读！
  • 运动习惯：爱运动的孩子，学习也不用愁

情绪认知：做情绪的主人

  • 情绪冲动的生理来源：杏仁核
  • 了解情绪的种类： 快乐、悲伤、厌恶、恐惧和愤怒
  • 倾听和同理心：建立感同身受的能力
  • 家长的情绪稳定与自我和解

专注力：专注此刻，静待花开

  • 从“发现式学习”到“掌控式学习”的过度

  • “默认模式网络”与“背侧注意网络”的切换

    • 适当的运动
    • 制定合理计划，并执行
    • 保持良好的精神状态

  • 专注力曲线，让人保持适度刺激水平进入专注力专区

    • 降温：运动改善人紧张状态
    • 升温：家长给予适当的互动刺激

  • 尝试建立长时间专注的场景（心流体验），帮助大脑进化习惯40分钟的上课习惯

    • 做喜欢的活动：弹钢琴，画画
    • 减少外界刺激，让孩子自主保持专注的态度

感知觉记忆

  • 视觉、听觉和主动复述可以增加记忆效率

  • 定期整理学习的内容，能够建立结构化记忆，帮助增加记忆容量

  • 科学根据记忆曲线更新记忆，从短期记忆变成长期记忆

  • 分段记忆代替整段记忆，降低记忆难度

自律

  • 自律来自对父母的爱和赞扬的渴求

  • 自律在身体和大脑疲劳时候，会削弱，保持充足的睡眠对自律性很重要

  • 建立正确的边界和规则意识
